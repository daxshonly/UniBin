# UniBin – Chrome Extension

Save any browser tab to Appwrite with a single click.

---

## Tech Stack

- **Frontend**: React 18 + TypeScript + Tailwind CSS (Vite)
- **Backend**: Appwrite Cloud (Web SDK v16)
- **Extension**: Manifest V3

---

## Step 1 — Appwrite Setup

### 1.1 Create a Project

1. Go to [cloud.appwrite.io](https://cloud.appwrite.io) and sign in.
2. Click **Create Project**, give it a name (e.g., `UniBin`), and note the **Project ID**.

### 1.2 Add a Web Platform

1. In your project, go to **Overview → Add a Platform → Web**.
2. Set the hostname to `chrome-extension://YOUR_EXTENSION_ID`.
   - You'll get the extension ID after loading the unpacked extension in Step 4.
   - For now you can temporarily use `*` or come back to set the exact ID.

### 1.3 Create the Database

1. Go to **Databases → Create Database** (e.g., name it `unibin_db`). Note the **Database ID**.
2. Inside the database, click **Create Collection** (e.g., `saved_tabs`). Note the **Collection ID**.

### 1.4 Add Collection Attributes

Add the following attributes to the collection (exact names required):

| Attribute Key | Type             | Required | Notes                          |
|---------------|------------------|----------|--------------------------------|
| `url`         | String (2048)    | ✅       |                                |
| `title`       | String (512)     | ✅       |                                |
| `platform`    | String (64)      | ✅       | e.g., "YouTube", "Reddit"      |
| `tags`        | String[] (Array) | ❌       | Item size: String (64)         |
| `savedAt`     | String (64)      | ✅       | ISO 8601 timestamp             |

### 1.5 Set Collection Permissions

For development/personal use, go to **Settings → Permissions** on your collection:
- Add a **Any** role with **Create** permission (or restrict to an authenticated user role).

## Step 2 — Configure `src/appwrite.ts`

Open `src/appwrite.ts` and replace the placeholder values:

```ts
export const APPWRITE_CONFIG = {
  endpoint: "https://cloud.appwrite.io/v1",
  projectId: "YOUR_PROJECT_ID",       // ← Appwrite Project ID
  databaseId: "YOUR_DATABASE_ID",     // ← Database ID
  collectionId: "YOUR_COLLECTION_ID", // ← Collection ID
} as const;
```

## Step 3 — Install & Build

```bash
# Install dependencies
npm install

# Production build → outputs to /dist
npm run build

# Optional: Vite dev server for UI iteration (no chrome.tabs, uses mock data)
npm run dev
```

---

## Step 4 — Load the Extension in Chrome

1. Open Chrome and navigate to `chrome://extensions/`.
2. Enable **Developer mode** (top-right toggle).
3. Click **Load unpacked** and select the `/dist` folder generated by the build.
4. The UniBin icon will appear in your toolbar. **Pin it** for easy access.

### Get your Extension ID

After loading, you'll see a string like `abcdefghijklmnopqrstuvwxyzabcdef` under the extension name. Copy it and go back to **Appwrite → Platform** to set the exact hostname:
```
chrome-extension://abcdefghijklmnopqrstuvwxyzabcdef
```

---

## Step 5 — Development Workflow

When you make changes to `src/`:

```bash
npm run build
```

Then in `chrome://extensions/`, click the **↻ Reload** button on the UniBin card.

> **Tip**: You can use `npm run dev` to iterate on UI in a browser tab. The app detects the absence of `chrome.tabs` and falls back to a mock YouTube tab so the full UI renders correctly.

## Platform Detection

The extension automatically detects these platforms from the URL and shows a color-coded badge:

| Platform       | Badge Color |
|----------------|-------------|
| YouTube        | Red         |
| Reddit         | Orange      |
| X / Twitter    | Sky Blue    |
| GitHub         | Violet      |
| LinkedIn       | Blue        |
| Medium/Substack| Green       |
| Stack Overflow | Amber       |
| Everything else| Slate (Web) |

To add more platforms, edit `src/utils/platform.ts` — just add an entry to `PLATFORM_MAP`.
